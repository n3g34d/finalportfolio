Iterative Design

This project involves iteratively designing a webpage for tech startup, Slai

Overview

While enrolled in CSCI1300 at Brown, I've learned that the process of web design is repetitive. The websites we view online are the results of drafts, feedback, and numerous iterations. As a group, three of my classmates and I worked together to attempt this process for tech startup, Slai. Together we researched Slai's mission, created mockups, gathered peer feedback, and conducted user testing to ultimately put together a website that we presented to Slai founders. This project provided us valuable experience with website creation and user friendly design.

Conclusion

This project gave me a thorough introduction to the iterative web design process. Our group enjoyed learned a lot from each, peers, and user testing regarding optimal web design for this tech startup. Following our completion of user testing, a member of our group emailed the startup with our findings. We were very pleased to recieve a response from the founders and happy to hear they liked our seamless integration of the text editor sandbox.

Takeaways:

  • Multiple iterations are key to designing a seamless project
  • Feedback and critique regarding web design can be contradicting
  • Participating in user testing is a great way to observe users iteract with your design
